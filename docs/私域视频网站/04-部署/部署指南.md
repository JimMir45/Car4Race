# 部署指南

## 1. 部署概述

### 1.1 部署架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                                   服务器                                      │
│                                                                              │
│  ┌───────────────────────────────────────────────────────────────────────┐  │
│  │                        Docker Compose Network                          │  │
│  │                                                                        │  │
│  │   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐           │  │
│  │   │    Nginx     │    │  Go Backend  │    │    MinIO     │           │  │
│  │   │  (Port 80)   │───→│  (Port 8080) │───→│ (Port 9000)  │           │  │
│  │   │              │    │              │    │ (Port 9001)  │           │  │
│  │   │  反向代理     │    │  API 服务    │    │  对象存储     │           │  │
│  │   │  静态资源     │    │  JWT 认证    │    │  文件管理     │           │  │
│  │   └──────────────┘    └──────────────┘    └──────────────┘           │  │
│  └───────────────────────────────────────────────────────────────────────┘  │
│                                       │                                      │
│  ┌────────────────────────────────────┴──────────────────────────────────┐  │
│  │                         Volume Mounts (宿主机持久化)                    │  │
│  │                                                                        │  │
│  │   ./data/db/              → SQLite 数据库                              │  │
│  │   ./data/minio/           → MinIO 对象存储                             │  │
│  │                             ├── courses/{id}/intro/   # 课程介绍       │  │
│  │                             └── courses/{id}/resource/ # 课程资源      │  │
│  └────────────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 系统要求

| 项目 | 最低配置 | 推荐配置 |
|-----|---------|---------|
| CPU | 1 核 | 2 核 |
| 内存 | 1 GB | 2 GB |
| 磁盘 | 10 GB | 50 GB（视课程文件大小） |
| 系统 | Linux（内核 3.10+） | Ubuntu 22.04 / CentOS 8+ |
| Docker | 20.10+ | 24.0+ |

### 1.3 端口要求

| 端口 | 用途 | 说明 |
|-----|------|------|
| 80 | HTTP 服务 | Nginx 入口（生产环境） |
| 443 | HTTPS 服务 | 可选，通过 Nginx 反向代理 |
| 8080 | Backend API | Go 后端服务（内部） |
| 9000 | MinIO S3 API | 对象存储服务（内部） |
| 9001 | MinIO Console | MinIO 管理控制台（可选暴露） |

## 2. 快速部署

### 2.1 使用 Docker 部署（推荐）

#### 步骤 1：准备目录结构

```bash
# 创建数据目录
mkdir -p /opt/hpa/data
mkdir -p /opt/hpa/static/notes
mkdir -p /opt/hpa/static/courses
mkdir -p /opt/hpa/static/covers

# 设置权限
chmod -R 755 /opt/hpa
```

#### 步骤 2：创建配置文件

```bash
# 创建配置文件
cat > /opt/hpa/config.yaml << 'EOF'
server:
  port: 8080
  mode: release  # debug / release

database:
  path: /app/data/hpa.db

jwt:
  secret: your-jwt-secret-key-change-this
  expire_hours: 168  # 7 days

sms:
  provider: aliyun  # aliyun / tencent
  access_key_id: your-access-key-id
  access_key_secret: your-access-key-secret
  sign_name: 你的签名
  template_code: SMS_123456789

security:
  user_rate_limit_seconds: 15
  guest_max_online: 50
  download_max_count: 3
  download_expire_hours: 168  # 7 days

member:
  threshold: 2000  # 年消费门槛
  announce_days: 3  # 公告展示天数
EOF
```

#### 步骤 3：拉取并运行容器

```bash
# 拉取镜像
docker pull your-registry/hpa:latest

# 运行容器
docker run -d \
  --name hpa \
  --restart always \
  -p 8080:8080 \
  -v /opt/hpa/config.yaml:/app/config.yaml:ro \
  -v /opt/hpa/data:/app/data \
  -v /opt/hpa/static/notes:/app/static/notes:ro \
  -v /opt/hpa/static/courses:/app/static/courses:ro \
  -v /opt/hpa/static/covers:/app/static/covers:ro \
  your-registry/hpa:latest
```

#### 步骤 4：验证部署

```bash
# 检查容器状态
docker ps | grep hpa

# 查看日志
docker logs -f hpa

# 测试接口
curl http://localhost:8080/api/v1/categories
```

### 2.2 使用 Docker Compose 部署（推荐）

#### docker-compose.yml

```yaml
version: '3.8'

services:
  # MinIO 对象存储服务
  minio:
    image: minio/minio:latest
    container_name: car4race-minio
    restart: unless-stopped
    command: server /data --console-address ":9001"
    environment:
      - MINIO_ROOT_USER=${MINIO_ACCESS_KEY:-car4race}
      - MINIO_ROOT_PASSWORD=${MINIO_SECRET_KEY:-car4race123}
    volumes:
      - ./data/minio:/data
    ports:
      - "9001:9001"  # MinIO Console（可选暴露）
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Go 后端服务
  backend:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: car4race-backend
    restart: unless-stopped
    environment:
      - ENV=production
      - PORT=8080
      - DB_PATH=/app/data/car4race.db
      - JWT_SECRET=${JWT_SECRET:-change-this-in-production}
      - MINIO_ENDPOINT=minio:9000
      - MINIO_ACCESS_KEY=${MINIO_ACCESS_KEY:-car4race}
      - MINIO_SECRET_KEY=${MINIO_SECRET_KEY:-car4race123}
      - MINIO_BUCKET=${MINIO_BUCKET:-car4race}
      - MINIO_USE_SSL=false
      - SMS_PROVIDER=${SMS_PROVIDER:-aliyun}
      - SMS_ACCESS_KEY=${SMS_ACCESS_KEY:-}
      - SMS_SECRET_KEY=${SMS_SECRET_KEY:-}
      - SMS_SIGN_NAME=${SMS_SIGN_NAME:-}
      - SMS_TEMPLATE_ID=${SMS_TEMPLATE_ID:-}
    volumes:
      - ./data/db:/app/data
    depends_on:
      minio:
        condition: service_healthy
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/api/v1/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # Nginx 反向代理
  nginx:
    image: nginx:alpine
    container_name: car4race-nginx
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
    depends_on:
      - backend
      - minio
```

#### 启动命令

```bash
# 1. 创建数据目录
mkdir -p data/db data/minio

# 2. 创建环境配置文件
cp .env.example .env
# 编辑 .env 文件，修改必要的配置

# 3. 启动所有服务
docker-compose up -d

# 4. 查看状态
docker-compose ps

# 5. 查看日志
docker-compose logs -f

# 6. 访问 MinIO Console（可选）
# 浏览器打开 http://your-server:9001
# 用户名: car4race (或 MINIO_ACCESS_KEY 的值)
# 密码: car4race123 (或 MINIO_SECRET_KEY 的值)
```

## 3. 配置说明

### 3.1 完整配置项

```yaml
# config.yaml 完整配置

server:
  port: 8080                    # 服务端口
  mode: release                 # 运行模式：debug / release
  read_timeout: 30              # 读超时（秒）
  write_timeout: 30             # 写超时（秒）

database:
  path: /app/data/hpa.db        # SQLite 数据库路径

jwt:
  secret: your-secret-key       # JWT 签名密钥（必须修改！）
  expire_hours: 168             # Token 有效期（小时）

sms:
  provider: aliyun              # 短信服务商：aliyun / tencent
  # 阿里云短信配置
  access_key_id: ""
  access_key_secret: ""
  sign_name: ""
  template_code: ""
  # 腾讯云短信配置（如使用腾讯云）
  # secret_id: ""
  # secret_key: ""
  # sdk_app_id: ""
  # sign_name: ""
  # template_id: ""

security:
  user_rate_limit_seconds: 15   # 用户接口调用间隔（秒）
  guest_max_online: 50          # 游客最大在线 IP 数
  download_max_count: 3         # 每次购买最大下载次数
  download_expire_hours: 168    # 下载链接有效期（小时）

member:
  threshold: 2000               # 会员触发门槛（元）
  expire_months: 12             # 会员有效期（月）
  announce_days: 3              # 新会员公告展示天数

static:
  notes_dir: /app/static/notes      # 笔记目录
  courses_dir: /app/static/courses  # 课程目录
  covers_dir: /app/static/covers    # 封面图片目录
```

## 4. Nginx 反向代理

### 4.1 基础配置

```nginx
# /etc/nginx/conf.d/hpa.conf

upstream hpa_backend {
    server 127.0.0.1:8080;
    keepalive 32;
}

server {
    listen 80;
    server_name your-domain.com;

    # 强制 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    # SSL 证书
    ssl_certificate /etc/nginx/ssl/your-domain.com.pem;
    ssl_certificate_key /etc/nginx/ssl/your-domain.com.key;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;

    # 日志
    access_log /var/log/nginx/hpa_access.log;
    error_log /var/log/nginx/hpa_error.log;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript;
    gzip_min_length 1000;

    # 前端静态文件缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        proxy_pass http://hpa_backend;
        proxy_cache_valid 200 30d;
        add_header Cache-Control "public, max-age=2592000";
    }

    # API 请求
    location /api/ {
        proxy_pass http://hpa_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 其他请求
    location / {
        proxy_pass http://hpa_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 4.2 大文件下载配置

```nginx
# 课程下载专用配置
location /api/v1/download/ {
    proxy_pass http://hpa_backend;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;

    # 大文件下载优化
    proxy_buffering off;
    proxy_request_buffering off;
    proxy_read_timeout 3600s;  # 1小时超时
    proxy_send_timeout 3600s;

    # 限速（可选）
    # limit_rate 10m;  # 限制 10MB/s
}
```

## 6. 数据备份

### 6.1 自动备份脚本

```bash
#!/bin/bash
# /opt/hpa/backup.sh

BACKUP_DIR="/opt/hpa/backups"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/hpa_backup_$DATE.tar.gz"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 停止容器（可选，保证数据一致性）
# docker stop hpa

# 备份数据
tar -czf $BACKUP_FILE \
  /opt/hpa/data \
  /opt/hpa/config.yaml

# 启动容器
# docker start hpa

# 清理 7 天前的备份
find $BACKUP_DIR -name "hpa_backup_*.tar.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_FILE"
```

### 6.2 设置定时任务

```bash
# 编辑 crontab
crontab -e

# 每天凌晨 3 点执行备份
0 3 * * * /opt/hpa/backup.sh >> /var/log/hpa_backup.log 2>&1
```

### 6.3 恢复数据

```bash
# 停止容器
docker stop hpa

# 解压备份
tar -xzf /opt/hpa/backups/hpa_backup_20250115_030000.tar.gz -C /

# 启动容器
docker start hpa
```

## 8. 常见问题

### 8.1 容器启动失败

**问题**：容器启动后立即退出

**排查**：
```bash
# 查看退出日志
docker logs hpa

# 检查配置文件
cat /opt/hpa/config.yaml

# 检查文件权限
ls -la /opt/hpa/
```

**常见原因**：
1. 配置文件格式错误
2. 数据目录权限不足
3. JWT Secret 未设置

### 8.2 数据库锁定

**问题**：SQLite 数据库被锁定，无法写入

**解决**：
```bash
# 检查是否有其他进程访问数据库
fuser /opt/hpa/data/hpa.db

# 重启容器
docker restart hpa
```

### 8.3 下载速度慢

**问题**：课程文件下载速度很慢

**解决**：
1. 检查服务器带宽
2. 优化 Nginx 配置（关闭 proxy_buffering）
3. 考虑使用 CDN

### 8.4 短信发送失败

**问题**：验证码发送失败

**排查**：
```bash
# 检查短信配置
grep -A 10 "sms:" /opt/hpa/config.yaml

# 查看相关日志
docker logs hpa | grep -i sms
```

**常见原因**：
1. AccessKey 配置错误
2. 短信签名未审核通过
3. 短信模板未审核通过
4. 账户余额不足

## 9. 版本升级

### 9.1 升级步骤

```bash
# 1. 备份数据
/opt/hpa/backup.sh

# 2. 拉取新镜像
docker pull your-registry/hpa:v1.1.0

# 3. 停止旧容器
docker stop hpa

# 4. 删除旧容器（保留数据卷）
docker rm hpa

# 5. 启动新容器
docker run -d \
  --name hpa \
  --restart always \
  -p 8080:8080 \
  -v /opt/hpa/config.yaml:/app/config.yaml:ro \
  -v /opt/hpa/data:/app/data \
  -v /opt/hpa/static/notes:/app/static/notes:ro \
  -v /opt/hpa/static/courses:/app/static/courses:ro \
  -v /opt/hpa/static/covers:/app/static/covers:ro \
  your-registry/hpa:v1.1.0

# 6. 执行数据库迁移（如有）
docker exec hpa ./hpa migrate

# 7. 验证服务
curl http://localhost:8080/api/v1/system/queue-status
```

### 9.2 回滚步骤

```bash
# 1. 停止新容器
docker stop hpa
docker rm hpa

# 2. 恢复数据（如果新版本修改了数据）
tar -xzf /opt/hpa/backups/hpa_backup_xxx.tar.gz -C /

# 3. 启动旧版本
docker run -d \
  --name hpa \
  ... \
  your-registry/hpa:v1.0.0
```
